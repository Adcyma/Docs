The Preferences page lets you set formatting rules for how Adcyma creates and displays user identity information. These settings keep things consistent across all new users and align with your naming conventions and email policies.

## Configurable Format Options

Use the dropdown menus to select your preferred format for each attribute:

**User Principal Name (UPN)**
How the primary login identifier is structured for new users.

**Display Name Format**
How user names appear in the system and in communications.

**Email Address Format**
How email addresses are constructed for new users.

**Primary Domain**
The domain used when generating email addresses and UPNs for new users. Select from the domains available in your Entra configuration.

## Advanced Name Settings

The Advanced Name Settings section gives you finer control over how Adcyma handles special cases in names:

- **Whitespace in Names** - Whether spaces in names are kept, removed, or replaced
- **Special Characters** - How accented and non-ASCII characters are handled
- **Hyphens in Names** - Whether hyphens in hyphenated names are kept or removed
- **Duplicate Names** - How Adcyma resolves conflicts when two users would generate the same UPN or email

## Preview Settings

Use the preview section to test how your format settings will look in practice. Enter a sample first name, last name, middle name, and ID to see the generated UPN, display name, and email in real time.

:::info Important Note
These formatting preferences **only apply to newly created users** and will not modify the format of existing user accounts. Current users will retain their existing UPN, email, and display name formats until manually updated.
:::

After making your selections, click **Save Changes** to apply them.

## Tips

Set these preferences early in your Adcyma deployment so all new users follow consistent naming patterns from the start. Make sure the formats you choose work with your existing email domains and directory structures, and that they'll still make sense as your organization grows.
